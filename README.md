## ğŸš€ Quick Start

### Prerequisites

- Node.js 18+ 
- MongoDB database
- npm or yarn package manager

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/your-username/blogged.git
   cd blogged
   ```

2. **Install dependencies**
   ```bash
   npm install
   ```

3. **Set up environment variables**
   Create a `.env.local` file in the root directory:
   ```env
   # Database
   DATABASE_URL="mongodb://localhost:27017/blogged"

   # Authentication
   NEXTAUTH_URL="http://localhost:3000"
   NEXTAUTH_SECRET="your-secret-key-here"

   # Optional: Demo credentials (UI only)
   NEXT_PUBLIC_DEMO_EMAIL="admin@example.com"
   NEXT_PUBLIC_DEMO_PASSWORD="password"

   # Optional: Media/CDN
   IMAGEKIT_URL=""

   # Optional: API base URL
   NEXT_PUBLIC_API_BASE_URL=""

   # Optional: CMS URL for redirects
   NEXT_PUBLIC_CMS_URL="http://localhost:3000"
   ```

4. **Set up the database**
   ```bash
   npx prisma generate
   npx prisma db push
   ```

5. **Seed the database with demo data**
   ```bash
   npm run seed
   ```

6. **Start the development server**
   ```bash
   npm run dev
   ```

7. **Open your browser**
   Navigate to [http://localhost:3000](http://localhost:3000)

### Demo Credentials

After running the seed script, you can sign in with:
- **Email:** admin@example.com
- **Password:** password

## âš™ï¸ Configuration

### Environment Variables

| Variable | Description | Required | Default |
|----------|-------------|----------|---------|
| `DATABASE_URL` | MongoDB connection string | Yes | - |
| `NEXTAUTH_URL` | NextAuth.js base URL | Yes | - |
| `NEXTAUTH_SECRET` | NextAuth.js secret key | Yes | - |
| `NEXT_PUBLIC_DEMO_EMAIL` | Demo user email | No | - |
| `NEXT_PUBLIC_DEMO_PASSWORD` | Demo user password | No | - |
| `IMAGEKIT_URL` | ImageKit CDN URL | No | - |
| `NEXT_PUBLIC_API_BASE_URL` | API base URL | No | Same origin |
| `NEXT_PUBLIC_CMS_URL` | CMS URL for redirects | No | - |

### Database Schema

The application uses MongoDB with the following main models:

- **User**: User accounts and authentication
- **Content**: Blog posts and articles
- **Comment**: User comments on content
- **Reaction**: User reactions (likes, etc.)
- **Rating**: Content ratings
- **Bookmark**: User bookmarks
- **StudyProgress**: Learning progress tracking
- **StudyPlan**: Learning plans and goals
